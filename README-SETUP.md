1) Firebase
  - This project includes a lightweight Firebase scaffold at `src/services/firebase.js`.
  - I populated `FIREBASE_CONFIG` in `src/config.js` using the values you provided.
  - To enable Firebase fully:
    a) Install firebase:
       npm install firebase
    b) Add your appId (from Firebase console) into `src/services/firebase.js` `firebaseConfig.appId` or into `src/config.js`.
    c) Restart the Expo dev server.

2) Google OAuth client IDs
  - Open `src/config.js` and paste the following values from the Google Cloud Console:
    - expoClientId: (use when testing via Expo proxy)
    - iosClientId
    - androidClientId
    - webClientId
  - Register redirect URIs in Google Cloud Console:
    - For Expo dev with proxy: use the URI generated by `makeRedirectUri({ useProxy: true })` (it looks like https://auth.expo.io/@your-username/your-project-slug)
    - For web/local: add http://localhost:19006 (or your local dev port) and your production origin as needed.

3) Secure storage
  - The app prefers `expo-secure-store` for persisting user data. To enable it:
    npm install expo-secure-store

4) Quick install commands
  npm install firebase
  npm install expo-secure-store

5) Restart Expo
  - After installing the native packages, restart the dev server and clear cache:
    expo start -c
